Galeriku — Upload & Media Flow v2 (Scalable)

Dioptimasi untuk concurrent users, batch operations, dan caching

Masalah & Solusi

MASALAH: Presigned URL per file
Batch 50 foto = 50 API calls untuk presign
SOLUSI: Batch presign endpoint
1 API call → presign max 20 files sekaligus. Reduce round trips 20x.
MASALAH: Metadata save per file
50 upload = 50 DB inserts = 50 connections
SOLUSI: Batch metadata insert
Client kumpulkan completed uploads, kirim batch setiap 5 files atau setiap 3 detik. 1 DB transaction per batch.
MASALAH: Signed URL per thumbnail
Album 200 foto × 10 users = 2000 signed URL generations
SOLUSI: R2 Public Bucket untuk thumbnails + CDN
Thumbnails di public bucket terpisah (low-res, no privacy risk). Served via Cloudflare CDN. Original files tetap private + signed URL.
MASALAH: No caching layer
Setiap view = direct hit ke R2
SOLUSI: Cloudflare Custom Domain untuk R2
R2 bucket di-attach ke custom domain → auto Cloudflare CDN caching. Thumbnails cached at edge. Originals via signed URL tetap aman.

R2 Dual-Bucket Strategy
🔒 galeriku-private
• Original photos (full resolution)
• Original videos
• Access via signed URL only
• URL expiry: photo 1hr, video 4hr
Key: media/{album_id}/{media_id}/original.{ext}
🌐 galeriku-thumbnails
• Thumbnails only (400px WebP, ~30KB)
• Public bucket + custom domain
• Cloudflare CDN cached
• No signed URL needed
Key: {media_id}/thumb.webp
⚠️ Unguessable media_id (uuid) = security through obscurity + low-res only
Thumbnails: CDN cached → near-zero latency, unlimited concurrent viewers   |   Originals: Private + signed URL → secure, per-request access control
Viewing — Optimized for Concurrent Users
Album Grid (thumbnails)
1. Server fetch media list (DB query)
2. Return media metadata + CDN thumbnail URL
3. No signed URL generation needed
4. Thumbnails served from CDN edge
5. Infinite scroll, 20 per page
6. ISR (Incremental Static Regen) — album page cached, revalidated on upload
Full View (original)
1. User clicks media → request signed URL
2. 1 signed URL per click (not batch)
3. Permission check per request
4. Prefetch next/prev for navigation
5. Client cache signed URL until expiry
Video Streaming
1. Signed URL (4hr expiry)
2. HTML5 <video> native player
3. R2 supports Range headers
4. Browser handles buffering
5. No transcoding needed
